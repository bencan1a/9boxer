## Special Cases & Ambiguous Terms

### Performance vs. Desempe√±o vs. Rendimiento

**Use "Desempe√±o"** for employee/human performance:
- ‚úÖ "Alto Desempe√±o" (High Performance - employees)
- ‚úÖ "Historial de Desempe√±o" (Performance History)

**Use "Rendimiento"** only for mechanical/technical performance:
- ‚ùå Not typically used in 9Boxer context

### Delete vs. Clear vs. Remove

**Delete (Eliminar)**: Permanent removal
- Used for: Permanently removing data

**Clear (Limpiar)**: Temporary removal, can be restored
- Used for: Clearing filters, clearing selections

**Remove (Quitar/Remover)**: General removal
- Used for: Removing from lists, removing filters

### Position vs. Posici√≥n vs. Puesto

**Position (Posici√≥n)**: Location in 9-box grid
- ‚úÖ "Position 1-9" (Grid position)
- ‚úÖ "Distribution by Position"

**Puesto**: Job title/role (NOT used in 9Boxer)
- ‚ùå Not used - we use "rol" if needed

### Manager vs. Gerente vs. Director

**Manager (Gerente)**: Direct supervisor
- ‚úÖ Used for immediate supervisor

**Director**: Higher-level management
- ‚úÖ "Exclude Directors+" (specifically directors)

**VP (VP)**: Vice President
- ‚úÖ Kept as "VP" in both languages

### Exclude vs. Excluir vs. Filter

**Exclude (Excluir)**: Remove from consideration
- Used for: Removing specific employees or groups

**Filter (Filtrar)**: Narrow down display
- Used for: Selecting subset to view

Both can result in fewer employees displayed, but intent differs.

## Pluralization Rules

### English
- Most nouns: add "s" ‚Üí employee**s**, change**s**
- Special cases handled by i18next: `_other` suffix

### Spanish
- Consonant ending: add "es" ‚Üí nivel ‚Üí nivel**es**
- Vowel ending: add "s" ‚Üí empleado ‚Üí empleado**s**
- Gender agreement: modificado/modificada/modificados/modificadas

### Implementation
Use i18next pluralization:
```json
{
  "employee": "empleado",
  "employee_other": "empleados"
}
```

## Contextual Usage Guidelines

### Formal vs. Informal Tone

**9Boxer uses a professional, formal tone:**
- Use "usted" forms in Spanish (implied)
- Commands use formal imperative: "Seleccione" not "Selecciona"
- Exception: Casual error messages may use informal: "¬°Ups!"

### Gender-Neutral Language

**Prefer gender-neutral terms when possible:**
- ‚úÖ "Empleado" (neutral/masculine as default)
- ‚úÖ "Gerente" (neutral)
- Avoid unnecessary gender-specific forms

### Title Case vs. Sentence Case

**English**: Use Title Case for buttons, menu items
- "Import Data", "Apply Changes"

**Spanish**: Use Sentence case with initial capital
- "Importar datos", "Aplicar cambios"
- Exception: Menu items may use Title Case: "Importar Datos"
